Welcome To InstaTok
  • Relations:

    We have 8 Relational Entity,

    1. Media: <Table that holds Information about each piece of media posted on service>
    2. Playlist: <A group of user selected media>
    3. Tags: <Describing entities that are attached to media>
    4. User: <Entities to represent a persons User account and its details>
    5. Comments: <Entities that users can add to media to depict their reactions>
    6. Explore Page: <The table that depicts the relationships >
    7. Downloading: <Table that logs what users download what media>
    8. Linked To: <What media is put in what playlist and the other way around>


  • Queries:

    The following Describes what each Query is expected to do:

    1. Query1: <Inner join on attributes of Media and Comments>
    2. Query2: <Count the number of comments posted for a given piece of media>
    3. Query3: <Selects all playlists which have a video media file in them using a subquery>
    4. Query4: <Displays all media and the number of comments each has if it is greater then 2 with grouping>
    5. Query5: <Displays all users who have posted any comments using a right outer join>
